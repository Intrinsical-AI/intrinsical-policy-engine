# Artifact Rendering Pipeline

```
plan.json
   ↓ (load)
ArtifactRenderer._load_plan
   ↓
build_artifact_context(plan)
   ↓ (audit injection, active roles/evidence)
Filtered templates
   ↓ (Jinja render + copy)
rendered artifacts (template tree)
```

## Scope

- The renderer only renders templates and copies static files.
- Integrity and traceability artifacts are generated by the export pipeline:
  `manifest.md`, `_metadata/trace.json`, `_metadata/actions.json`,
  `CHECKSUMS.sha256`, and `fingerprint.json`.

## Determinism & Integrity Invariants

- Template traversal is sorted for deterministic output across platforms.
- Content determinism is enforced by export manifests and checksums, not by the renderer.

## Roles & Evidence Filtering

1. Flags → `ARTIFACT_SELECTOR.get_active_roles` decides which directories under `evidence/templates/` are relevant.
2. Actions metadata → `get_active_evidence_paths` restricts to evidence referenced by selected actions.
3. `ARTIFACT_SELECTOR.should_include_evidence_path` combines role + evidence filters for deterministic inclusion.

## When to Update This Module

- Changing template filtering rules (roles, evidence, internals).
- Modifying rendering context injection.
- Introducing new template categories or extensions.
